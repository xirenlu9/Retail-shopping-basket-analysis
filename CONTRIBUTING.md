# 贡献指南 (Contributing Guide)

感谢您对本项目的关注！我们欢迎各种形式的贡献，包括修复 Bug、增加新功能、改进文档或提出建议。

## 如何贡献

1.  **提交 Issue**: 如果您发现了 Bug 或有新功能的想法，请先在 [Issue 列表](https://github.com/your-username/your-repo/issues) 中搜索是否已存在。如果不存在，请新建一个 Issue。
2.  **Fork 仓库**: 将本项目 Fork 到您自己的 GitHub 账号下。
3.  **创建分支**: 为您的修改创建一个新的分支 (例如 `feature/new-analysis` 或 `fix/bug-id`)。
4.  **进行修改**: 编写代码、更新文档，并确保代码风格与现有代码保持一致。
5.  **提交 Pull Request**: 将您的分支推送到 GitHub，并向主仓库提交 Pull Request。

## 代码规范

-   请保持代码简洁易读。
-   注释尽量清晰，特别是涉及算法逻辑的部分。
-   提交 PR 前请确保脚本可以正常运行。

## 行为准则

参与本项目即表示您同意遵守我们的 [行为准则 (CODE_OF_CONDUCT.md)](CODE_OF_CONDUCT.md)。

---
再次感谢您的贡献！
